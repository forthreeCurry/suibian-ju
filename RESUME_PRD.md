# 「随便聚」AI 智能聚餐决策平台 - 产品需求文档

## 一、产品战略定位

### 1.1 产品愿景
**产品名称**：随便聚（Suibian Ju）  
**Slogan**：谁再喊随便，今晚谁买单！  
**核心使命**：用 AI 算法 + 群体决策机制，3 分钟解决多人聚餐的世纪难题

### 1.2 市场洞察与机会

#### 目标场景
- **高频刚需**：朋友聚餐、同事聚会、家庭聚餐等社交场景
- **决策痛点**：
  1. "随便"死循环：人人都说"都行"，结果无人满意
  2. 社交包袱：预算、忌口等敏感信息难以公开表达
  3. 组织者压力大：承担繁重的找店比对工作 + 决策失误被吐槽的风险
  4. 沟通效率低：群聊信息碎片化，反复讨论无结果

#### 市场机会
- 社交聚餐决策市场规模巨大，但缺乏专业化工具
- 现有解决方案（大众点评、美团）侧重单一个体决策，缺少群体协作功能
- AI 推荐技术成熟，可应用于多约束条件下的群体偏好匹配

### 1.3 产品定位
**智能群体决策工具**：通过结构化偏好收集 + 多约束 AI 算法，在 3 分钟内让聚餐人群达成餐厅共识

### 2.3 数据库设计

#### 核心表结构

**1. rooms（房间表）**
```sql
- id: UUID (主键)
- short_code: TEXT (6 位分享码，唯一索引)
- host_id: TEXT (发起人 member_id)
- status: TEXT (状态机：waiting → voting → calculating → finished)
- name: TEXT (聚餐名称，如"周五火锅局")
- scene: TEXT (场景类型：instant 即时聚 / planned 提前聚)
- max_members: INT (人数上限，默认 10)
- date_time: TIMESTAMPTZ (聚餐时间)
- location_strategy: TEXT (位置策略：smart 智能商圈 / manual 手动选择)
- budget_mode: TEXT (预算模式：anonymous 匿名 / unified 统一)
```

**2. members（成员表）**
```sql
- id: UUID (主键)
- room_id: UUID (外键关联 rooms)
- nickname: TEXT (昵称)
- avatar_url: TEXT (随机 Emoji 头像)
- joined_at: TIMESTAMPTZ (加入时间)
```

**3. preferences（偏好表 - 隐私保护核心）**
```sql
- id: UUID (主键)
- member_id: UUID (唯一外键，级联删除)
- budget: TEXT (人均预算，如 "120")
- dietary_restrictions: JSONB (忌口列表，如 ["香菜", "海鲜过敏"])
- taste_likes: JSONB (口味偏好结构化数据)
- departure_location: JSONB (出发地 {"lat", "lng", "address"})
- transport_mode: TEXT (交通方式：subway/drive/bike/walk)
```

**4. results（AI 推荐结果表）**
```sql
- id: UUID (主键)
- room_id: UUID (外键关联 rooms)
- recommendations: JSONB (Top N 餐厅推荐数组)
- weather_info: JSONB (天气数据快照)
- created_at: TIMESTAMPTZ
```

#### 关键设计亮点
- **Row Level Security (RLS)**：MVP 阶段允许匿名用户所有操作，降低使用门槛
- **Realtime 实时订阅**：房间状态、成员偏好、推荐结果变更实时推送
- **JSONB 灵活存储**：偏好数据采用 JSONB 格式，支持复杂结构化数据

## 三、核心功能模块详解

### 3.1 用户流程与功能架构

#### 主流程
```
发起聚餐/加入房间 → 填写出发地 → 匿名预算/忌口 → 口味卡片测试 →
```

### 3.2 核心功能模块

#### 模块一：房间创建与分享系统

**功能描述**：
- 支持两种场景：即时聚（线下碰头）/ 提前聚（线上预热）
- 6 位数字房间码，便于微信分享
- 自动分配 Emoji 头像，增强趣味性

**产品亮点**：
- **极简创建流程**：仅需填写昵称，3 秒完成房间创建
- **分享码机制**：相比 URL 更易传播，降低加入门槛
- **状态机管理**：waiting → voting → calculating → finished，清晰的状态流转

**技术实现**：
```typescript
// 房间创建核心逻辑
const handleCreate = async () => {
  const shortCode = generateShortCode(); // 6 位数字码
  const avatar = AVATARS[Math.floor(Math.random() * AVATARS.length)];
  
  // 插入房间 + 成员，事务保证一致性
  const { data: room } = await supabase
    .from("rooms")
    .insert({ short_code: shortCode, host_id: "pending", ... })
    .select("id").single();
    
  const { data: member } = await supabase
    .from("members")
    .insert({ room_id: room.id, nickname, avatar_url: avatar })
    .select("id").single();
    
  // 更新 host_id 为实际成员 ID
  await supabase.from("rooms").update({ host_id: member.id }).eq("id", room.id);
};
```

#### 模块五：AI 推荐引擎（核心算法）

**功能描述**：
- 多约束条件匹配：位置交集、预算下限、忌口黑名单、口味偏好
- 天气因子：根据天气状况推荐室内/室外餐厅
- 个性化推荐理由：针对不同用户展示不同卖点

**算法逻辑**：
```typescript
// 推荐数据结构
type Recommendation = {
  name: string;        // 餐厅名称
  cuisine: string;     // 菜系
  avgPrice: number;    // 人均消费
  rating: number;      // 评分
  tags: string[];      // 标签
  reason: string;      // 个性化推荐理由
  matchScore: number;  // 匹配度分数
};

// 算法伪代码
function calculateMatchScore(restaurant, preferences) {
  let score = 0;
  
  // 1. 预算匹配（权重 30%）
  if (restaurant.avgPrice <= userBudget) score += 30;
  
  // 2. 口味标签匹配（权重 40%）
  const likedTags = intersection(restaurant.tags, userLikes);
  score += likedTags.length * 10;
  
  // 3. 忌口过滤（一票否决）
  if (hasDietaryConflict(restaurant, userRestrictions)) return 0;
  
  // 4. 天气因子（权重 15%）
  if (weather.isRainy && !restaurant.indoor) score -= 20;
  if (weather.isHot && !restaurant.hasAC) score -= 15;
  
  // 5. 评分加成（权重 15%）
  score += restaurant.rating * 3;
  
  return score;
}
```

- 每家餐厅包含：菜系、人均价、评分、标签、室内/空调属性

## 四、产品亮点与创新点

### 4.1 核心创新

- **解决问题**：传统筛选条件繁琐，用户决策成本高
- **实现方式**：8 张美食主题卡片 + 三档评分

- **用户价值**：在复杂限制中找到最优解，提升整体满意度

## 七、风险与挑战

### 7.1 技术风险
- **餐厅数据准确性**：依赖第三方 API，存在数据更新延迟风险
- **高并发场景**：节假日高峰期可能出现性能瓶颈

### 7.2 产品风险
- **用户习惯培养**：需要教育用户使用新工具替代传统讨论方式
- **网络效应**：需要达到临界用户量才能形成自传播

### 7.3 应对策略
- **数据兜底**：建立本地餐厅缓存，API 异常时降级展示
- **性能优化**：引入 CDN、数据库索引、查询优化
- **增长策略**：种子用户运营、社交裂变激励
