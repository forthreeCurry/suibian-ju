# 「随便聚」产品需求文档 (PRD)

## 1. 产品概述
* **产品名称**：随便聚
* **核心定位**：一个多人聚餐场景下的智能决策产品。通过结构化偏好收集与多约束算法计算，让聚餐人群在3分钟内达成餐厅共识。
* **产品形态**：移动端 H5 / 微信小程序（便于社交裂变与快速分享）。

## 2. 场景与痛点 (为什么要解决这个问题)
1. **“随便”死循环**：餐厅选择，人人说“都行”，结果聚餐时又没有真正满意。
2. **“开口难”的社交包袱**：出于社交礼仪，不好意思提要求，预算、忌口、饮食偏好难以公开表达。
3. **发起人的至暗时刻**：组局者不仅要承担极其繁重的“找店-比对”脑力劳动，还要承担“选错店被吐槽”的巨大决策压力。
4. **效率低下**：群里讨论消息刷屏，信息碎片化，且流程繁琐。
   * *传统流程痛点*：组织者提议 -> 讨论地点 -> 询问偏好 -> 多个美食APP反复横跳搜索 -> 截图发群 -> 沉默/零星讨论 -> 比对+说服+决策。
   * *问题本质*：多人场景下的协调选择成本，远高于找餐厅本身。

## 3. 产品介绍与核心场景
「随便聚」旨在减少聚餐组织的沟通和决策成本，给出能兼顾全员需求的餐厅选择。核心支持以下两种场景：

### 场景一：线上提前选店铺 (组局预热)
1. 发起人设定聚餐基本信息（商圈、时间等）。
2. 发送聚餐邀请（分享卡片至微信群）。
3. 成员点开应用，完成“位置 + 匿名预算 + 口味偏好”的信息填写。
4. AI综合分析计算，输出 Top3 方案及针对每个人的推荐理由。
5. 成员对店铺进行投票。
6. 确定最终店铺，一键定位/导航。

### 场景二：面对面即时碰头 (线下破冰)
1. 一群人在线下不知吃什么，打开「随便聚」。
2. 开启“碰一碰”或扫码加入线下聚餐组。
3. 各自填写预算、口味和偏好。
4. AI结合当前共同定位，综合分析输出附近的 Top3 方案。
5. 大家查看选项，最终决定去哪家。

## 4. 产品亮点与核心功能模块

### 4.1 偏好推算（免打字体验）
* **功能描述**：用户不需要自己进行繁琐的分类筛选，通过提供美食图片或选项卡片做选择。
* **AI逻辑**：AI根据用户的选项构建潜在口味偏好模型。例如：用户选择麻辣火锅、炭火烤肉，拒绝清蒸鲈鱼和高档西餐，AI推断出用户需要“重口味、爱肉类、喜欢热闹有烟火气的就餐氛围”。

### 4.2 信息保护与匿名提交
* **功能描述**：预算、特殊饮食要求（如忌口、过敏）等涉及隐私的信息，部分用户不好意思公开表达。
* **隐私机制**：将信息提交给「随便聚」，AI只做最基础的分析判断。在保护用户个人隐私的情况下，让用户不再因为社交顾虑而隐藏真实需求（其他人看不到你的底线预算和特殊忌口）。

### 4.3 责任分担（AI背锅侠）
* **功能描述**：引入“AI推荐+群体决策”机制，弱化组织者的个人责任。
* **心理学设计**：如果对选择的餐厅不满意，将吐槽的炮火对准“AI背锅侠”，极大降低组织者的心理负担。后续可指引用户完成对AI的评价，达到“边吐槽边优化”的正向循环。

### 4.4 多约束条件及个性化推荐
* **算法引擎**：同时处理位置交集、预算下限、忌口黑名单、口味偏好等多约束条件，在众多限制中找到最优解。
* **千人千面**：根据不同用户给出的偏好，在同一个餐厅结果下，展示独属于该用户的个性化推荐理由（如：对A说“有你爱吃的辣”，对B说“在你预算范围内”）。
